# AI Agent Trading Platform вҖ” Complete UI Prompt

## Project Overview
Build a **Next.js 14 (App Router) + Tailwind CSS** frontend for an AI multi-agent stock research platform focused exclusively on **Indian stock markets (NSE)**. The platform sends a ticker symbol to a Python backend (`POST /anyalse`), receives structured analyst reports + a bull-bear debate + final verdict, and presents them in a clean research terminal UI.

---

## Design System

**Palette вҖ” "Editorial White"**
```
Background:        #F8F8F6  (warm off-white, never pure white)
Surface cards:     #FFFFFF
Borders:           #E4E4E0
Primary text:      #111110  (near-black)
Secondary text:    #888884
Muted label:       #BBBBB7
Accent вҖ“ BUY:      #16A34A  (green-600)
Accent вҖ“ SELL:     #DC2626  (red-600)
Accent вҖ“ HOLD:     #D97706  (amber-600)
Active sidebar:    #111110  (inverted вҖ” black bg, white text)
```

**Typography**
- Body + UI: `Inter`
- Tickers, verdicts, mono data: `JetBrains Mono`
- Install both via `next/font/google`

**Aesthetic:** WSJ meets Notion. Clean, editorial, high information density. Sharp corners everywhere. No gradients. No shadows (except `shadow-sm` on cards). Borders do the heavy lifting. The only color on the entire page is the verdict badge.

## Data & API Contracts

### Ticker Data (client-side, no API needed)
```js
// /public/nse-tickers.json
// Array of objects вҖ” build from NSE500 list
[
  { "symbol": "RELIANCE", "name": "Reliance Industries Ltd" },
  { "symbol": "TCS", "name": "Tata Consultancy Services Ltd" },
  { "symbol": "INFY", "name": "Infosys Ltd" },
  // ... 500 entries
]
```
Use `fuse.js` for fuzzy client-side search. User **must select from dropdown** вҖ” free-text submission is disabled.

### Backend API
```
POST /anyalse
Body: { "ticker": "RELIANCE" }

Response:
{
  "ticker": "string",
  "news_report": "string",        // markdown
  "technical_report": "string",   // markdown
  "fundamental_report": "string", // markdown
  "market_report": "string",      // markdown
  "sector_report": "string",      // markdown
  "investment_debate": {
    "bull_thesis": "string",       // markdown
    "bear_thesis": "string",       // markdown
    "speaker_history": ["bull", "bear", "manager"],
    "last_speaker": "manager",
    "final_decision": {
      "decision": "HOLD | BUY | SELL",
      "rationale": "string"
    }
  },
  "research_verdict": {
    "decision": "HOLD | BUY | SELL",
    "rationale": "string"
  },
  "status": "string"
}
```

- Full width, height `48px`
- Background `#111110`, text `#F8F8F6`, Inter 14px medium

## Constraints & Notes
1. **No pure black (`#000000`) or pure white (`#FFFFFF`) as backgrounds** вҖ” always use the warm off-white / card white distinction
2. **The only colors on the page are the three verdict accents** вҖ” everything else is zinc/warm-gray
3. All corners are **sharp (no border-radius)** except the verdict decision word which can have `rounded-sm`
4. **Mobile is not a priority** вҖ” design for `min-width: 1024px` desktop
5. Store API response in `sessionStorage` keyed by ticker вҖ” so sidebar switching doesn't re-fetch
6. If `status` from API is not `"success"`, show an inline error state in the main panel (not a modal)
7. The `speaker_history` array drives debate order вҖ” render bubbles in that sequence, pulling content from `bull_thesis`, `bear_thesis`, or `final_decision.rationale` based on speaker name
